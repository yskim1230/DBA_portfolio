# Start Here

## 1. 요약
저는 8년차 DBA로 온프레미스 MSSQL, MySQL, MariaDB 운영과 클라우드 NCP Cloud DB for MySQL 운영 전환 업무를 수행했습니다.
운영을 감으로 하지 않고 절차와 증적 중심으로 표준화하는 방식으로 일합니다.
이 저장소는 제가 만든 런북, 스크립트, 로그, 정량 지표를 기반으로 재현 가능한 형태로 정리한 포트폴리오입니다.

## 3. 저장소 읽는 순서
1) 온프레미스에서 운영 기본기 백업 복구 HA 보안 튜닝을 확인
2) 클라우드에서 전환 과정 인벤토리 표준화 통합 이관 관측 자동화를 확인
3) 각 문서는 배경 절차 검증 결과 리스크 롤백 순서로 구성

## 4. 기술 범위
온프레미스
- MSSQL 운영 백업 복구 이중화 장애대응 성능튜닝 보안감사
- MySQL MariaDB 운영 MHA 복제 기반 HA 백업 복구

클라우드
- NCP Cloud DB for MySQL 운영
- VPC 기반 접근통제 트러블슈팅
- 로그 수집 파이프라인 Export to Object Storage 기반
- 관측 체계 PMM, Active Session 샘플링, Slow 로그 일일 리포트

## 5. 보안과 비밀정보 처리
이 문서는 공개 저장소 기준으로 작성되었습니다.
사내 도메인 계정 키 티켓 시스템 식별자는 마스킹하거나 제거했습니다.
운영 원문은 사내 정책에 따라 별도 보관합니다.
